# CardVault

CardVault is a marketplace and collection management platform for physical trading cards. The project focuses on enabling collectors to buy, sell, trade, and track real-world cards while building collections, earning achievements, and participating in a trusted community.

The application is built with SolidStart and TypeScript and is designed with a domain-driven architecture that separates business concepts from storage and presentation concerns.

---

# Vision

Most card marketplaces focus exclusively on transactions.

CardVault aims to combine:

* Marketplace functionality
* Collection tracking
* Trading between users
* Reputation systems
* Achievements and badges
* Set completion progress
* Verified card ownership

The goal is to create a platform that serves both collectors and traders.

---

# Core Features

## Marketplace

Users can:

* Browse available listings
* Search for specific cards
* Create sell orders
* Purchase cards from other users
* Track transaction history

---

## Collection Management

Users can:

* Register cards they own
* Organize collections
* Track set completion
* Monitor collection value
* Discover missing cards

---

## Trading System

Users can:

* Offer card-for-card trades
* Offer card + cash trades
* Negotiate exchanges
* Track trade history

---

## Achievements and Badges

Collectors earn achievements for milestones such as:

* First completed purchase
* First successful sale
* First successful trade
* Completing a set
* Owning rare cards
* Reaching collection size milestones

---

## Reputation System

Users build reputation through:

* Successful sales
* Successful purchases
* Successful trades
* Positive feedback
* Account longevity

---

## Verification

To reduce fraud, card listings may require:

* Front photo
* Back photo
* Verification code generated by the platform
* Condition declaration

Future versions may support manual verification and escrow workflows.

# Domain Model

## User

Represents a platform member.

Properties:

* id
* username
* email
* reputation
* registrationDate

---

## Card

Represents the official definition of a trading card.

Properties:

* id
* name
* game
* expansion
* collectionNumber
* officialImage

Examples:

* Charizard Base Set
* Blue-Eyes White Dragon LOB
* Black Lotus Alpha

---

## Card Copy (Physical Instance)

Represents a specific physical card owned by a user.

Properties:

* id
* cardId
* ownerId
* condition
* frontImage
* backImage

This distinction allows multiple users to own copies of the same card.

---

## Listing

Represents a sale offer.

Properties:

* id
* sellerId
* cardCopyId
* price
* status
* createdAt

---

## Purchase

Represents a completed transaction.

Properties:

* id
* buyerId
* listingId
* amount
* purchaseDate

---

## Trade

Represents an exchange between users.

Properties:

* id
* sourceUserId
* targetUserId
* status

Future versions may support multiple cards and mixed-value offers.

---

## Collection

Represents a user-managed collection.

Properties:

* id
* userId
* name

---

## Badge

Represents an achievement that can be earned.

Properties:

* id
* name
* description
* icon

# Development Philosophy

The project follows a domain-first approach:

```text
Business Rules
      ↓
Domain Model
      ↓
Database
      ↓
API
      ↓
User Interface
```

This keeps business logic independent from storage technologies and UI frameworks.
